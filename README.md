<H1>SEQUEL FANTASY RPG</H1>

<h2>Objetivo</h2>
Criar uma estrutura de banco de dados sólida para um universo de RPG de fantasia, aplicando boas práticas de modelagem relacional e gerenciamento de dados. O projeto tem como objetivo unir aprendizado e criatividade, permitindo explorar conceitos de banco de dados enquanto se desenvolve um mundo fictício repleto de personagens, mascotes e elementos de fantasia.

<h2>Sobre o projeto</h2>
Projeto de banco de dados em SQL que modela um universo de RPG de fantasia utilizando um sistema relacional. O banco armazena informações sobre jogadores, mascotes mágicos, mundos, skins, armas e seus relacionamentos, permitindo consultas e gerenciamento eficiente dos dados do jogo.

O mundo é composto por diferentes espécies de mascotes, como Dragões, Fadas, Lobos Lunares e Brotos, cada uma com características e habilidades únicas. Além disso, os jogadores podem possuir skins com diferentes estilos e temas, enriquecendo a personalização dos personagens.

<h2>Funcionalidades</h2>
<ul>
  <li>Cadastro de jogadores</li>
  <li>Cadastro de mascotes e espécies</li>
  <li>Cadastro de skins, estilos e temas</li>
  <li>Cadastro de mundos, caracteristicas e temas</li>
  <li>Relacionamentos entre entidades</li>
  <li>Consultas utilizando JOINs</li>
  <li>Gerenciamento de dados com comandos SQL</li>
</ul>

<h2>Conceitos Aplicados</h2>
<ul>
 <li>Modelagem Entidade-Relacionamento (MER) </li>
 <li>Chaves Primárias (Primary Key) </li>
 <li>Chaves Estrangeiras (Foreign Key) </li>
 <li>Relacionamentos entre tabelas </li>
 <li>JOINs </li>
 <li>Normalização de dados </li>
 <li>Comandos DDL e DML </li>
 </ul>
